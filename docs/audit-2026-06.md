## 2. Stack technique

Le site est **100 % statique** : zéro build, zéro framework, zéro dépendance runtime hors Google Fonts.

| Couche | Technologie |
|---|---|
| Markup | HTML5 (9 pages) |
| Styles | CSS3 — 1 fichier, BEM, custom properties |
| Scripts | JS vanilla ES5 — 2 fichiers |
| Police | Google Fonts (Inter + Space Grotesk) via `@import` CDN |
| Météo | Open-Meteo Forecast API (gratuit, sans clé) |
| Géocodage | Open-Meteo Geocoding API (gratuit, sans clé) |
| Reverse geocoding | Nominatim / OpenStreetMap (gratuit, sans clé) |
| Hébergement | Nginx — fichiers statiques |

## 5. Issues restantes

### 5.1 Majeures

| ID | Fichier(s) | Description | Impact |
|---|---|---|---|
| M1 | Tous les HTML | Header, nav (9 liens) et footer copiés-collés dans les 9 pages (~52 lignes × 9 = ~468 lignes dupliquées, soit ~25 % du HTML) | Toute modification nav/footer = 9 fichiers |
| M2 | `news.html` | Les 4 liens "Lire la suite →" pointent vers `href="#"` | UX cassée, scroll to top |
| M3 | Tous les footers | "Mentions légales" et "Conditions d'utilisation" → `href="#"` | Obligation légale belge/UE non satisfaite |
| M4 | `js/meteo.js:464` | En-tête `User-Agent` personnalisé pour Nominatim silencieusement ignoré par les navigateurs (header interdit) — risque de rate-limit par Nominatim | Conformité ToS Nominatim |

### 5.2 Mineures

| ID | Fichier(s) | Description |
|---|---|---|
| m1 | Tous | Pas de favicon → 404 sur chaque page |
| m2 | Tous | Pas de balises Open Graph → prévisualisations sociales génériques |
| m3 | Racine | Pas de `robots.txt` ni `sitemap.xml` |
| m4 | `css/style.css:1` | `@import` Google Fonts render-blocking (remplacer par `<link>` dans `<head>`) |
| m5 | 7 pages HTML | `style="padding-top: 0;"` inline répété → devrait être une classe CSS |
| m6 | `biblio.html` | `card__text` (BEM de `.card`) utilisé dans `.biblio-item` (couplage inter-blocs) |
| m7 | `js/meteo.js:208` | Score seeing peut dépasser 10 (ex: jet 400 km/h → score 12/10) — formule non bornée |
| m8 | `js/meteo.js:36` | Phase lunaire simplifiée (époque fixe + période moyenne) — dérive légère sur phases intermédiaires |
| m9 | `album.html` | Items galerie sans `role="img"` ni `aria-label` |
| m10 | Plusieurs pages | Emojis dans les `<h2>` non marqués `aria-hidden` |
| m11 | Tous les footers | Icône Facebook = lettre "f" en texte brut (pas un vrai logo SVG) |
| m12 | `js/meteo.js:146` | `formatTime()` utilise `Date.getHours()` — dépend du fuseau horaire du navigateur, pas de la localisation météo |

## 7. Audit CSS (style.css)

### Points forts
- ✅ 18 variables CSS (`:root`) couvrant couleurs, typographie, radius, shadows, transitions
- ✅ BEM cohérent (`block__element--modifier`) sur l'ensemble des composants
- ✅ Mobile-first, 2 breakpoints (640 px, 1024 px)
- ✅ Animations : `float`, `spin`, `pulse`, transitions fluides

### Points faibles
- ⚠️ `.card`, `.weather-card`, `.news-card`, `.featured-card` partagent ~60 % de styles identiques (pas de classe de base commune)
- ⚠️ `style="padding-top:0"` inline dans 7 pages (devrait être `.section--flush`)
- ❌ Pas de `prefers-color-scheme` (thème sombre forcé)
- ❌ `@import` Google Fonts en première ligne → render-blocking

### meteo.js (~633 lignes)

| Fonctionnalité | État | Notes |
|---|---|---|
| Fetch météo | ✅ | Open-Meteo, 11 variables horaires + daily, try/catch |
| Traitement données | ✅ | `processOpenMeteo()` → index heure courante → 8 cartes |
| Phase lunaire | ✅ | Époque fixe + période synodique — précision ±quelques heures |
| Seeing | ✅ | `(jet/40)+(cloud/100)+(humidity/100)` — attention : non borné (cf. m7) |
| Géocodage ville | ✅ | Debounce 300 ms, état loading + erreur |
| Géoloc GPS | ✅ | `navigator.geolocation` + Nominatim, messages d'erreur dédiés |
| Historique localisations | ✅ | localStorage, max 5, sans doublons, FIFO |
| `showLoading()` | ✅ | Corrigé → `display:'flex'` (C2) |
| `showSearchError()` | ✅ | Corrigé → `createElement` + `textContent` (M6) |
| User-Agent Nominatim | ⚠️ | Header interdit → ignoré silencieusement (M4) |
| Heure lever/coucher | ⚠️ | `Date.getHours()` dépend du fuseau local (m12) |
| Score seeing | ⚠️ | Peut dépasser 10/10 pour jet stream > 400 km/h (m7) |

#### Algorithme seeing

```
score = (wind_speed_250hPa / 40) + (cloud_cover / 100) + (humidity / 100)
```

| Score | Label | Couleur |
|---|---|---|
| ≤ 2,5 | Excellent | `#22c55e` |
| ≤ 4,5 | Bon | `#84cc16` |
| ≤ 6,5 | Moyen | `#eab308` |
| ≤ 8,5 | Médiocre | `#f97316` |
| > 8,5 | Mauvais | `#ef4444` |

#### Sources de données météo

| API | Endpoint | Clé requise | Quota |
|---|---|---|---|
| Open-Meteo Forecast | `api.open-meteo.com/v1/forecast` | Non | 10 000 req/jour |
| Open-Meteo Geocoding | `geocoding-api.open-meteo.com/v1/search` | Non | Partagé |
| Nominatim | `nominatim.openstreetmap.org/reverse` | Non | 1 req/s (politesse) |

#### Variables météo récupérées

```
hourly: cloud_cover, cloud_cover_low, cloud_cover_mid, cloud_cover_high,
        relative_humidity_2m, dew_point_2m, temperature_2m,
        wind_speed_10m, wind_gusts_10m, wind_speed_250hPa
daily:  sunrise, sunset
```

## 11. Ce qui fonctionne bien

- Architecture CSS propre : BEM + design tokens + mobile-first
- Aucune clé API exposée (toutes les APIs sont sans authentification)
- `rel="noopener noreferrer"` systématique sur les liens externes
- `meteo.js` bien structuré : IIFE, ES5, gestion d'erreurs par scénario, `localStorage` avec try/catch
- Back-to-top avec listener `{ passive: true }`
- Messages d'erreur géolocalisation en français (codes 1/2/3 mappés)
- Fallback lune calculée même en cas d'échec API météo
